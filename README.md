A physical quest paper system for Minecraft servers featuring 31 objective types, dynamic quest generation, real-time progress tracking, and intuitive GUI management.

---

## ‚ú® Key Features

- **Physical Quest Papers** ‚Äì Quest items exist in player inventories
- **31 Objective Types** ‚Äì Combat, building, collection, survival, movement, and more
- **Multi-Objective Quests** ‚Äì Complete multiple objectives in any order or sequentially
- **Real-Time Progress** ‚Äì BossBar, ActionBar, or Chat tracking
- **Flexible Rewards** ‚Äì XP, money, custom items, and command execution
- **Quest Conditions** ‚Äì Level requirements, costs, permissions, and PlaceholderAPI support
- **Random Generation** ‚Äì Auto-generate infinite quests with configurable templates
- **Quest Loot System** ‚Äì Quests drop from chests and mob kills
- **Interactive GUIs** ‚Äì Browse, create, and edit quests in-game
- **Leaderboards** ‚Äì Track player progress with PlaceholderAPI integration

## üöÄ Installation

1. Download `SoapsQuest.jar` from [Releases](https://github.com/AlternativeSoap/SoapsQuest/releases)
2. Place in your `plugins/` folder
3. Restart the server
4. Configure `plugins/SoapsQuest/quests.yml`
5. Give quests with `/sq give <player> <quest>`

---

## ‚ö° Quick Start

Create your first quest in `plugins/SoapsQuest/quests.yml`:

```yaml
zombie_slayer:
  display: "&aZombie Slayer"
  tier: common
  difficulty: easy
  objectives:
    - type: kill
      target: ZOMBIE
      amount: 10
  reward:
    xp: 100
    money: 50
```

Give it to a player:

```
/sq give Steve zombie_slayer
```

Players can also browse and accept quests:

```
/sq list
/sq browse
```
