SoapsQuest V1.0.2 Changelog
============================

Release date: May 2026

Overview
--------
Logging controls, quest queue fixes, active-limit condition fix, and documentation corrections.

Quest queue & progress
----------------------
- Fixed multiple copies of the same quest completing from a single action (event snapshot).
- Different quest IDs can be Active at the same time; duplicate IDs stay Queued until claim.
- Recurring daily/weekly papers register progress and join the queue.
- Active/Queued lore updates correctly after queue refresh.

Logging
-------
- Claim messages no longer spam console by default (was always INFO, not debug).
- config.yml: log-quest-completions (default false), log-admin-actions (default false).
- Console text now says "claimed quest" when logging is enabled.
- Queue verification and reward-editor INFO moved behind debug mode.

Conditions
----------
- active-limit now counts Active quest types in the queue, not all papers in playerdata.
- Extra copies of the same quest ID queue without counting against the limit.

Quality of life
---------------
- Removed dead "blocked" / "replaced" queue branches from /sq give and GUI give paths.

Documentation
-------------
- FAQ and Objectives: per-quest-type Active/Queued (removed incorrect max-active-quests reference).

Requirements (unchanged)
--------------------------
- Paper 1.21+, Java 21+, SoapsCommon
- Optional: Vault, PlaceholderAPI, MythicMobs
